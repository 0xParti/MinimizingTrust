Appendix C: Field Equations Cheat Sheet

A quick reference for the core equations that power zero-knowledge proof systems.

Schwartz-Zippel Lemma

The most important bound in the book.

For a non-zero polynomial p(X1​,…,Xn​) of total degree d over a field F:

r←FnPr​[p(r)=0]≤∣F∣d​

Consequence: Random evaluation catches cheating with probability ≥1−d/∣F∣.

Multilinear Extensions

Lagrange Basis Polynomial

For w∈{0,1}n:

Lw​(X)=i=1∏n​(wi​⋅Xi​+(1−wi​)(1−Xi​))

Property: Lw​(w)=1 and Lw​(b)=0 for b=w.

Multilinear Extension Formula

For f:{0,1}n→F:

f~​(X)=w∈{0,1}n∑​f(w)⋅Lw​(X)

Equality Polynomial

eq​(X,Y)=i=1∏n​(Xi​Yi​+(1−Xi​)(1−Yi​))

Property: eq​(a,b)=1 if a=b, else 0 (on hypercube).

Sum-Check Protocol

Dimensions: Vector size N=2n; protocol runs n rounds.

The Claim

Prove: H=b∈{0,1}n∑​g(b1​,…,bn​)

Round i Polynomial

Prover sends: si​(Xi​)=bi+1​,…,bn​∈{0,1}∑​g(r1​,…,ri−1​,Xi​,bi+1​,…,bn​)

Verifier Checks

  • Round 1: s1​(0)+s1​(1)=H
  • Round i>1: si​(0)+si​(1)=si−1​(ri−1​)
  • Final: query oracle at (r1​,…,rn​) to check sn​(rn​)=g(r1​,…,rn​)

Soundness

ϵ≤∣F∣n⋅d​

where n is the number of variables and d is the maximum individual degree. (More precisely: ∑i​di​/∣F∣ where di​ is the degree in variable i.)

Vanishing Polynomials

Over Roots of Unity

For domain H={1,ω,ω2,…,ωn−1} where ωn=1:

ZH​(X)=Xn−1

Property: ZH​(ωi)=0 for all i, and ZH​(r)=0 for r∈/H.

Over Boolean Hypercube

For proving a polynomial vanishes on {0,1}n, use the univariate identity:

Z{0,1}​(X)=X(X−1)

applied variable by variable in multilinear settings.

R1CS and QAP

R1CS Constraint

Dimensions: Matrices A,B,C are m×n; witness z is n×1; result is m×1.

For witness vector z=(1,x,w):

(A⋅z)∘(B⋅z)=C⋅z

where ∘ is entry-wise multiplication.

QAP Polynomial Identity

Define polynomials A(X),B(X),C(X) by interpolating constraint matrices.

The constraint system is satisfied iff:

A(X)⋅B(X)−C(X)=H(X)⋅ZH​(X)

where ZH​(X)=∏α∈H​(X−α) is the vanishing polynomial.

KZG Polynomial Commitments

Dimensions: Polynomial degree <D; SRS size D+1 elements.

Structured Reference String (SRS)

Secret τ; public: (g,gτ,gτ2,…,gτD)

Commitment

For f(X)=∑i​ci​Xi:

C=gf(τ)=i∏​(gτi)ci​

Evaluation Proof

To prove f(z)=v: (Prover knows f(X); Verifier knows C, z, v)

  1. Compute quotient: w(X)=X−zf(X)−v​
  2. Proof: π=gw(τ)

Verification (Pairing Check)

e(π,gτ⋅g−z)=e(C⋅g−v,g)

Equivalently:

e(gw(τ),gτ−z)=e(gf(τ)−v,g)

FRI Folding

Split Polynomial

For f(X)=fE​(X2)+X⋅fO​(X2):

  • fE​(Y): even coefficients
  • fO​(Y): odd coefficients

Folding with Challenge α

f1​(Y)=fE​(Y)+α⋅fO​(Y)

Property: deg(f1​)<deg(f)/2

Consistency Check

At query point x (where −x is its conjugate on the same coset), verify:

f1​(x2)=2f(x)+f(−x)​+α⋅2xf(x)−f(−x)​

This uses: fE​(x2)=2f(x)+f(−x)​ and fO​(x2)=2xf(x)−f(−x)​.

AIR (Algebraic Intermediate Representation)

Trace Polynomials

For a trace matrix with w registers and T timesteps, interpolate each column over domain H={1,ω,…,ωT−1}:

Pj​(ωi)=trace[i][j]

Transition Constraints

For a constraint "register 0 at next step equals f of current registers":

P0​(ωX)=f(P0​(X),P1​(X),…)

The shift ωX accesses "next row" values. Define constraint polynomial:

C(X)=P0​(ωX)−f(P0​(X),P1​(X),…)

Quotient Check

Valid trace iff C(X) vanishes on transition domain H′={1,ω,…,ωT−2}:

Q(X)=ZH′​(X)C(X)​

is a polynomial (not rational function).

Boundary Constraints

Pin inputs/outputs. For Pj​(ωk)=v:

X−ωkPj​(X)−v​

must be a polynomial.

PLONK

Gate Equation

QL​(X)⋅a(X)+QR​(X)⋅b(X)+QO​(X)⋅c(X)+QM​(X)⋅a(X)⋅b(X)+QC​(X)=0

on domain H={1,ω,ω2,…,ωn−1}.

Permutation Grand Product

Accumulator Z(X) satisfies:

Z(1)=1

Z(ωi+1)=Z(ωi)⋅(ai​+βσa​(ωi)+γ)(bi​+βσb​(ωi)+γ)(ci​+βσc​(ωi)+γ)(ai​+βωi+γ)(bi​+βk1​ωi+γ)(ci​+βk2​ωi+γ)​

Property: The product telescopes, so Z(ωn)=Z(1)=1 iff all copy constraints hold.

Quotient Check

All constraints satisfied iff there exists t(X) with:

(gate)+α⋅(permutation)=t(X)⋅ZH​(X)

Groth16

Public Input Combination

Given public inputs (z0​,z1​,…,zℓ​) where z0​=1:

vkx​=j=0∑ℓ​zj​⋅(vkIC​)j​

where (vkIC​)j​=g1γβAj​(τ)+αBj​(τ)+Cj​(τ)​​ are verification key elements.

Verification Equation

Given proof (πA​,πB​,πC​)∈G1​×G2​×G1​:

e(πA​,πB​)=?e(g1α​,g2β​)⋅e(vkx​,g2γ​)⋅e(πC​,g2δ​)

Verification cost: One MSM (size ℓ) + 3-4 pairings, independent of circuit size.

3 group elements: 128 bytes over BN254 (32 + 64 + 32 for G1​, G2​, G1​).

Lookup Arguments

Plookup Identity

For lookups f={f1​,…,fn​} and table t={t1​,…,td​}, let s=sort(f∪t).

i∏​(γ+fi​)⋅i∏​(γ(1+β)+ti​+βti+1​)⋅(1+β)n

=i∏​(γ(1+β)+si​+βsi+1​)

Property: Equality holds iff f⊆t.

LogUp Identity

For lookups f={f1​,…,fn​} into table t={t1​,…,td​} with multiplicities mj​:

i=1∑n​γ+fi​1​=j=1∑d​γ+tj​mj​​

Property: Equality holds iff each fi​∈t and mj​ counts occurrences correctly.

Soundness: By Schwartz-Zippel, equality holds with probability ≥1−(n+d)/∣F∣ over random γ.

Advantage: No sorting required; additive structure enables multi-table batching.

GKR Protocol

Dimensions: Layer i has Si​ gates; layer i+1 (inputs) has Si+1​ gates; k=log2​Si​.

Layer Reduction

For layered circuit with values V(i) at layer i:

V(i)(r)=p,q∈{0,1}k∑​add(i)(r,p,q)⋅(V(i+1)(p)+V(i+1)(q)) +mult(i)(r,p,q)⋅V(i+1)(p)⋅V~(i+1)(q)

Sum-Check Reduction

A claim about V(i)(r) reduces via sum-check to claims about V(i+1)(p∗) and V~(i+1)(q∗) for random p∗,q∗.

Soundness: Compound over d layers, each with O(logn) sum-check rounds.

Inner Product Argument (IPA)

The Claim

Prove ⟨a,b⟩=c for committed a.

Folding Step

Given challenge α:

a′=α⋅aL​+α−1⋅aR​ b′=α−1⋅bL​+α⋅bR​

Property: ⟨a′,b′⟩=⟨a,b⟩+α2L+α−2R

where L=⟨aL​,bR​⟩ and R=⟨aR​,bL​⟩.

Proof Size

O(logn) group elements after logn rounds.

Nova Folding

Relaxed R1CS

Standard R1CS: (A⋅z)∘(B⋅z)=C⋅z

Relaxed R1CS with scalar u and error E:

(A⋅z)∘(B⋅z)=u⋅(C⋅z)+E

A satisfying instance has u=1 and E=0.

Folding Two Instances

Given instances (u1​,E1​,z1​) and (u2​,E2​,z2​), with challenge r:

u=u1​+r⋅u2​ E=E1​+r⋅T+r2⋅E2​ z=z1​+r⋅z2​

where T is the "cross-term" computed by the prover.

Property: If both inputs satisfy relaxed R1CS, so does the folded instance.

Fiat-Shamir Transform

Challenge Derivation

ri​=H(transcript prefix including all previous messages)

Security Requirement

The hash must include:

  • The public statement x
  • All previous commitments C1​,…,Ci−1​
  • All previous challenges r1​,…,ri−1​

Quick Reference: What to Use When

Proving a sum over hypercube: Sum-check protocol

Encoding data as polynomial: Multilinear extension (hypercube) or Lagrange interpolation (roots of unity)

Binding prover to polynomial: KZG (trusted setup, constant size), FRI (transparent, log² size), IPA (no pairings, log size)

Checking polynomial identity on a domain: Quotient by ZH​(X)=Xn−1 for roots of unity

Checking table membership: Lookup argument (Plookup with sorting, LogUp without)

Verifying circuit layer-by-layer: GKR protocol with sum-check at each layer

Incremental computation: Nova folding (amortize SNARK cost across steps)

Eliminating interaction: Fiat-Shamir with complete transcript hashing
